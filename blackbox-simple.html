▶Purpose

Our mission is to legally investigate and observe online crimes, with emphasis on protecting children and animals. We do not intervene. We collect, verify, and compile evidence into structured case files for formal review and archival.

Current structure carries roleplay residue. To ensure sustainability and credibility, we propose a rework of roles, workflow, and recruitment to form a professional but simple investigative system.

▶Role Structure (Simplified & Realistic)
▶Oversight & Leadership
  • Director of Operations — Final authority, approves or rejects completed case files.
  • Oversight Committee — Supports the Director in reviewing and ensuring legality and standards.
▶Case Roles
  • Case Manager — Leads one case. Manages a team of 5–7 investigators.
  • Senior Investigator — Mentors new recruits.
  • Investigator — Collects and submits evidence (screenshots, IDs, context).
  • Analyst — Organizes and formats evidence; supports the Case Manager.
▶Support & Specialists
  • Recruitment Officer — Runs entry challenges and onboarding.
  • Technical Specialist — Provides OSINT, metadata, and technical support.
  • Archivist — Maintains case file archives.
▶Case Workflow
  1. Case Assignment — Oversight assigns a Case Manager, who forms a small team.
  2. Evidence Collection — Investigators collect:
    • Screenshots
    • User/Message IDs, channel references
    • Context/description for each piece
  3. Compilation — Case Manager reviews and builds a Case File using a clear format.
  4. Approval — Case File submitted to Director → approved (archived) or denied (returned for edits).
▶Case File Format (Consistent Template)
  1. Case ID / Name
  2. Case Manager
  3. Investigators Assigned
  4. Summary of Case (short description)
  5. Evidence Log
    • Item (screenshot / file / link)
    • Source (channel ID, user ID, message ID)
    • Context (why relevant)
    • Notes
  6. Conclusion / Recommendation
  7. Approval Status (signed by Director)

Tip: enforce this template with a form (Notion/Docs) so submissions are uniform.

▶Recruitment Process (Cicada‑Style)

Entry is by skill, not request. Challenges mirror required competencies:

  • Logic & analysis (riddles, problem solving)
  • Research skills (finding hidden info online)
  • Attention to detail (steganography / hidden messages)
  • Technical skills (basic encryption/decryption, metadata analysis)

After completion, recruits select a path: Investigator, Case Manager trainee, or Specialist. Each path has a proficiency exam.

▶Benefits of This Rework
  • Professional — Removes gamey roleplay feel; replaces with serious investigative structure.
  • Simple — Clear workflow that scales without bloat.
  • Consistent — Every case documented under the same template.
  • Exclusive — Recruitment by puzzles signals rigor and intent.
▶Archive Protocol
  1. Approved case files receive a unique immutable Case ID.
  2. Redacted public summary may be produced (optional, oversight‑approved).
  3. All source artifacts stored in read‑only archive with checksum notes.
  4. Access controlled by Archivist; audit log retained.
▶Closing Note

Designed to humanize process without diluting seriousness. Simple, clear, legal. Focus remains on protection through disciplined observation and documentation.
